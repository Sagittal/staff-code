Smart Position

the most recently used position is automatically applied if none is specified
don't manifest until they are needed (only apply to glyphs with ligatures to be vertically shifted by them)
persists until a new position is used (is 'sticky')
does not actually put the middle position on the stave; there is no zero position glyph in SMuFL, so we temporarily use a code point from the Sagittal range, but it has no actual effect so it shouldn't be emitted
works for the positions supplement

Smart Clef

assume treble clef even if no clef has been provided
change depending on the clef
clefs are smart and they stick until you change them (you can change from one to the other)
when a pitch code (e.g. C4) is used, position sticks based on the pitch, not the vertical offset; if you set a pitch like C4 and then change clefs, it should still be C4 in the new clef, not whichever note is at the same position as the previous clef's C4
in contrast, when a position code (e.g. up3) is used, position sticks based on the position offset, not the pitch
the autogenerated SMuFL codes for combining staff positions (stpsrs & stpslr) work exactly the same as up and dn (the latter are simply aliases for them)

the advance-to-end code works for glyphs with widths of exactly 24

the advance-to-end code can be adjusted with manual advance
the advance-to-end code can be adjusted with manual advance, when the manual advance amount is greater than the difference between the width of the glyph and the remaining stave

Smart Stave

automatically adds stave lines as needed, if smart stave has been asked for
does not add additional lines as needed, if only a manual stave has been asked for
can be turned off and on, and upon turning it off, it advances you to the end of the stave it has already laid down
provides leger lines automatically for notes and noteheads beyond ±5 staff positions
only puts leger lines on every other position (the ones that would have been lines)
provides multiple leger lines if the note is very beyond the staff
turns off leger lines when smart stave is off
provides leger lines of the smallest size which is wider than the glyph they are for, up to 48 lgln6
provides leger lines of the smallest size which is wider than the glyph they are for, up to 48 lgln11
provides leger lines of the smallest size which is wider than the glyph they are for, up to 48 lglnwd15

when smart stave is on, and a code is unrecognized, it first finishes the smart stave, and then for that final space (the one for separating it from other unrecognized codes, but nice to have regardless) it turns smart stave back on, to ensure the next code appears on top of stave
when smart stave is off, and a code is unrecognized, it behaves otherwise the same, with the same amount of spacing that is, but just no stave
